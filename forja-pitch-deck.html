02 / 12 Problema

Três fraturas estruturais no mercado de agentes IA:

  • LLMs esquecem tudo entre sessões — cada conversa começa do zero, sem contexto acumulado, sem aprendizado sobre o usuário ou o projeto.
  • Ferramentas atuais orquestram parcialmente, não integram — o dev monta cola entre SDKs, APIs e prompts; a inteligência fica fora do produto.
  • Agente custom = 6–12 meses de engenharia — memória, persona, segurança, fallback, mobile: cada camada exige time dedicado antes de gerar qualquer valor.
03 / 12 Solução

Um agente que lembra, aprende e respeita limites — sem construir nada.

🧠
Memória Episódica
Contexto persistente nativo cross-sessão. A Ayla lembra decisões, preferências e histórico de trabalho entre dispositivos.
🎯
Psicometria DISC / Eneagrama
Calibração de tom, ritmo e profundidade ao perfil cognitivo do usuário — menos atrito, mais resultado por interação.
🛡️
Veto Hierarchy
Hierarquia de autoridade programável. Root sobrescreve tudo; regras de equipe propagam automaticamente sem override acidental.
04 / 12 Produto

Um cérebro único em toda plataforma — local-first na CLI, remote-first no mobile.

Camada · CLI
Local-First
Node.js · runtime localhost · sem dependência de nuvem para uso offline. Latência de inferência, não de rede.
Camada · Mobile / Web
Remote-First
Angular · Ionic · Capacitor. Centrifugo WebSocket exclusivo (sem fallback SSE). Paridade cognitiva ~95% com a CLI.
Fonte de Verdade
GitHub
Toda alteração de persona, regra e config passa por PR — auditável, reversível, team-friendly.
Orquestração
Ayla via EC2 · Bedrock
Claude + Qwen. Tool-loop com budget de tokens, streaming, fallback fail-closed. Múltiplos agentes especializados em paralelo.
