08Papel no time · Backend Engineer

Sua função no ecossistema

Backend puro no início — sem precisar dominar IA. A curva de aprendizado em IA é gradual e estruturada. Stack: NestJS · Prisma · PostgreSQL (Neon) · TypeScript · AWS EC2 · Stripe.

Fase 0 · Semanas 1–2 · Sprint go-live URGENTE
Stripe webhook → provisionar org no Neon → ativar plano
Self-serve signup (email + GitHub OAuth)
Free tier enforced — 10 invocações/dia, 1 agente
Deploy EC2 estável + CI/CD automático
Health pipeline — /healthz + alertas p95 Neon
Testes e2e — cobertura mínima antes do go-live
Fase 1 · Mês 1–2 · Desacoplamento do server monolítico

O server.ts tem 6.500 linhas — tudo acoplado num único arquivo. É o maior risco de escala do ecossistema. Não precisa entender IA para fazer.

Extrair BillingModule — Stripe, planos, cotas
Extrair AuthModule — JWT ES256, rate limiter, sessions
Extrair AgentInvokeModule — bridge de invocação
Extrair ChannelModule — mensageria IRC + SSE
server.ts vira bootstrap — monta módulos, sem lógica
Testes por módulo — cobertura isolada e auditável
Fase 2 · Mês 3+ · Entrada no pipeline de agentes

Curva de IA gradual — você aprende o ecossistema pelo código antes de tocar em cognição.

Pipeline invoke_agent — entender o flow EC2 → agente → resposta
Observability por agente — latência, budget, erros por orgId
PLG features — referral tracking, activation funnel
Fundah backend — reposicionamento + BCM integração
Battle Arena — features open source
Melhoria de agentes — só após dominar o pipeline
O que ela NÃO toca agora: core/ayla/, corpus Laura, persona, vault, constituição — área exclusiva do Rafael. Protocolos obrigatórios: TDD-first · §422 · PRE_FLIGHT.
