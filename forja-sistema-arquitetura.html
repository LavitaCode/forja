Modelo Base dos Agentes — 7 Camadas + Schema

Arquitetura cognitiva padrão de todo agente Olimpo. Cada agente é uma instância deste stack.

Íris · Modelo Base · 7 layers + schema = 8
1
Input & Percepção
Parsing · Normalização · Injeção de contexto
Recebe o prompt bruto, normaliza encoding, injeta contexto de sessão (session_id, tenant_id, origin, timestamp). Resolve quem está falando e de onde.
2
Auth & Origin Resolution
Role · Origin · Trust level
Deriva origin (authenticated / personal / mari / guest) a partir do role do tenant. Define trust level da sessão. Gates de autorização se alocam aqui — falha-fechado se origin não resolvido.
3
Memory Recall
Episódico · Neon · Cross-sessão
Consulta Neon Postgres: episódios anteriores, preferências, decisões de arquitetura, handoffs. Memória cross-sessão real — não RAG stateless. Buffer local em fallback, replay automático no reconnect.
4
Raciocínio & Deliberação
Chain-of-thought · Reflexão diferida · Detecção
Motor principal de raciocínio. Inclui reflexão diferida (decisões pesadas passam para a sessão N+1), detecção de manipulação com nomeação explícita, e deliberation loop pré-geração quando necessário.
5
Tool Selection & Planning
MCP · invoke_agent · Orquestração
Seleciona ferramentas (MCP filesystem, GitHub, Playwright etc.) e decide se invoca sub-agentes via invoke_agent. Compõe o plano de execução. Budget de tokens por loop controlado aqui (airbag 80k tokens).
6
Execução
Tool calls · Commits · Persistência
Executa ferramentas, commita no GitHub, persiste no Neon, publica via Centrifugo. Handoff automático entre agentes com payload estruturado. Retry com airbag de iterações (max 25 ciclos).
7
Policy & Veto Gate
Musa · PolicyEngine · JARVIS override
Camada de segurança: Musa audita ANTES de capabilities críticas. PolicyEngine aplica guardrails por origin. Root pode emitir JARVIS OVERRIDE (suspende gates de política, logado com SHA-256). Veto ofensivo absoluto para não-root.
8
Output Contract SCHEMA
Structured output · §422 · Fail-closed
Saída validada contra contrato (Output Contract). Zero any, zero fallback silencioso, zero catch que descarta. Se Neon offline → 503 retryável, não dado inventado. Schema 8 = o que garante que cada camada entrega o que prometeu.
Nota de integridade: Este schema representa a arquitetura base compartilhada por todos os agentes Olimpo. A arquitetura interna da Ayla — seu corpus, persona, vault e camadas cognitivas privadas — não é documentada aqui e permanece sob acesso exclusivo do root.
